# Face Age Morphometrics

Predict chronological age from MRI-derived 3D facial morphometrics (GPA + EDMA) and evaluate
how the face-age signal compares with brain-age derived from the same T1 scan.

---

## Overview

Standard brain-age models predict a subject's brain age from structural MRI. This project
asks whether the *face* captured incidentally in a T1 head scan carries an independent
age signal. We extract 3D facial landmarks using the BioFace3D-20 MVCNN, compute
Generalised Procrustes Analysis (GPA) and Euclidean Distance Matrix Analysis (EDMA)
features, and train a Ridge regression model on the IXI dataset.

**Research hypotheses:**
- **H1** — Face morphometrics predict chronological age significantly better than chance.
- **H2** — Face-age gap correlates with brain-age gap across individuals (IXI cross-sectional).
- **H3** — Face-age variability across repeated scans (SIMON) is lower than brain-age variability.

## IXI dataset preparation

**Source:** [IXI Brain Development Dataset](https://brain-development.org/ixi-dataset/)
(563 T1 NIfTI scans, sites: Guys and Hammersmith Hospital, ages 20–86 yr, ~55% female)

**Step 1 — Build metadata and locked splits:**
```bash
python scripts/prepare_metadata.py
```
Creates `data/ixi_train.csv`, `data/ixi_val.csv`, `data/ixi_test.csv` using a
stratified split (site × sex × age-decade). Split sizes: train=335 / val=69 / test=72.
The split is locked — re-run with `--force` only if you intentionally want to reshuffle.

**Step 2 — Extract features:**
```bash
python scripts/extract_features.py --device mps --predict-num 5
```

Per-subject pipeline (checkpointed, resumable):
1. Load T1 NIfTI, reorient to RAS+
2. Histogram-match to IXI intensity template (SimpleITK)
3. Predict isosurface threshold (sklearn LinearRegression on intensity statistics)
4. Extract head mesh isosurface (VTK FlyingEdges3D)
5. Clean + hollow head mesh (PyMeshLab: remove disconnected components, AO hollowing)
6. Crop to face region, reorient to LSA space (PyMeshLab + PyVista)
7. Detect 20 facial landmarks (BioFace3D-20 MVCNN, MRE ≈ 1.61 mm on IXI)

Then for all splits (fit on train only — no leakage):

| Step | Method | Output |
|------|--------|--------|
| GPA | Iterative Procrustes superimposition + sklearn PCA | 9 components (99.6% variance) |
| EDMA | C(20,2) = 190 pairwise Euclidean distances | 190-dim vector |
| Combined | Concatenate GPA scores + EDMA distances | 199-dim feature vector |

Saved artefacts:
- `data/features/gpa_mean_shape.npy` — (20, 3) Procrustes mean shape (train only)
- `data/features/gpa_pca.joblib` — fitted sklearn PCA (train only)
- `data/features/{train,val,test}.npz` — feature arrays + metadata

## Model benchmarking

```bash
python scripts/benchmark_regressors.py
```

Six regressors evaluated with hyperparameters tuned by cross-validation on the train
set only. Val and test sets are never seen during tuning.

### IXI test set results (199-dim features, 9 GPA components)

| Model | Train MAE | Val MAE | Test MAE | Test RMSE | Test R² | Test r | Bias |
|-------|-----------|---------|----------|-----------|---------|--------|------|
| **Ridge** | **7.26** | **7.75** | **7.81** | **9.77** | **0.586** | **0.775** | −1.83 |
| Lasso | 7.56 | 7.79 | 8.06 | 10.04 | 0.562 | 0.760 | −1.84 |
| ElasticNet | 7.63 | 7.88 | 8.07 | 10.03 | 0.563 | 0.760 | −1.82 |
| SVR | 2.11 | 9.14 | 8.49 | 10.45 | 0.526 | 0.746 | −2.54 |
| GBM | 0.28 | 8.89 | 9.04 | 10.97 | 0.477 | 0.709 | −2.41 |
| MLP | 2.10 | 11.59 | 10.38 | 13.22 | 0.241 | 0.658 | +0.43 |

**Winner: Ridge regression.**

Rationale: with n=335 training subjects and d=199 features, the dataset strongly
favours low-variance linear models. SVR, GBM, and MLP all overfit severely
(train MAE 0.3–2.1 yr vs test MAE 8.5–10.4 yr). Ridge achieves the best test MAE
and the smallest train/test gap (0.55 yr), indicating good generalisation.

Per-sex breakdown on test set (Ridge):
- Males (n=33): MAE = 6.47 yr, RMSE = 7.97 yr
- Females (n=39): MAE = 8.95 yr, RMSE = 11.06 yr

Saved artefacts:
- `data/models/benchmark/ridge_model.joblib` — use for inference
- `data/models/benchmark/scaler.joblib` — IXI train StandardScaler (always `.transform()`, never `.fit_transform()` on new data)
- `data/models/benchmark/benchmark_results.csv` — full 6-model comparison table

## SIMON dataset preprocessing

**Source:** [SIMON dataset](https://www.mcgill.ca/bic/resources/simon)
(1 healthy male subject, 99 scans across 73 sessions, 36 scanners, 13 MRI models,
ages 29.69–46 yr, acquisition dates 2001–2019)

```bash
python scripts/extract_simon_landmarks.py --device mps --predict-num 5
```

The same pipeline as IXI is applied with two additional preprocessing steps required
for SIMON's scanner diversity:

1. **Intensity normalisation** — some scanners store raw float intensities in the
   millions (vs IXI's 0–4095 range). Scans where `max > 8190` are rescaled to 0–4095
   via 99th-percentile clipping before histogram matching and threshold prediction.

2. **Mesh decimation** — dense meshes (>200K faces) caused the BioFace3D MVCNN
   renderer to fail silently (RANSAC back-projection error ~15M mm). Face meshes are
   now decimated to ≤200K faces using quadric edge collapse (PyMeshLab) before
   landmark detection.

63 of 99 scans succeeded. Remaining failures fall into three categories:
- Head extraction failure (empty isosurface after AO hollowing) — 4 scans
- Face crop failure (empty mesh) — 4 scans
- Landmark detection failure (MVCNN RANSAC error, cause under investigation) — 28 scans

Predictions are run via `notebooks/07_predict_new_dataset.ipynb`, which applies the
IXI-trained Ridge model and scaler without refitting. All 99 sessions are included in
the output CSV with `NaN` for failed scans and a `prediction_status` column.

Output: `results/simon_face_age.csv`
Columns: `session_id`, `run`, `age`, `manufacturer`, `scanner_model`, `institution`,
`face_age_pred`, `error`, `abs_error`, `centroid_size`, `procrustes_dist`, `prediction_status`
